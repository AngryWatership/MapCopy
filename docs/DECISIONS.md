# DECISIONS.md
> Rationale log. Append-only. Prevents re-litigating settled choices.
> Each entry: what was decided, why, and what was rejected.

---

## D-001 — Abandon Python, rewrite as web app

**Decided:** Session 1
**Decision:** Port to Vanilla JS + GitHub Pages. Discard tkinter entirely.
**Reason:** Desktop-only, not shareable, no deployment path, no metrics possible.
**Rejected:** Electron (overkill), React (unnecessary complexity for v1).

---

## D-002 — Two-key-pair model, no cycling

**Decided:** Session 1
**Decision:** Every character requires exactly 2 keypresses. No multi-press cycling.
**Reason:** Confirmed by author. Multi-press was a misreading of the original code.
**Rejected:** T9-style cycling (was never the intent), timeout-based auto-commit.

---

## D-003 — 2D index formula

**Decided:** Session 1
**Decision:** `idx = row_off * 4 + col_off` where offsets are modular distances in the 4×3 grid.
**Reason:** Extends the original row-wise formula naturally to the vertical axis.
**Rejected:** Flat 12-key index (loses spatial intuition), lookup table (not generalisable).

---

## D-004 — Sign correction in formula

**Decided:** Session 1
**Decision:** `col_off = (col_B - col_A + 4) % 4` — second minus first.
**Reason:** Verified against three author-provided examples: `i→u=l`, `j→l=c`, `o→p=r`. Original Python had `(index - _index)` which is first minus second — a bug.
**Rejected:** Original sign (produces wrong characters).

---

## D-005 — 144-slot layout with zero duplicates

**Decided:** Session 1
**Decision:** Each key holds 12 unique characters. No character appears twice across the full 144-slot layout.
**Reason:** 16 slots were previously duplicated (home-row cross pairs repeated letters). Replaced with Ctrl shortcuts.
**Rejected:** Allowing duplicates as "convenience aliases" (adds confusion, wastes slots).

---

## D-006 — layout.json generated by script, never hand-edited

**Decided:** Session 1
**Decision:** `mapcopy_layout.py` is the single source of truth. `layout.json` is output.
**Reason:** Ensures verification runs on every change. Prevents silent inconsistencies.
**Rejected:** Editing JSON directly (no verification, error-prone).

---

## D-007 — Visual band order: 1, 0, −1

**Decided:** Session 1
**Decision:** Inside each key in training view, bands stack top-to-bottom as: row-above chars / same-row chars / row-below chars.
**Reason:** Spatially intuitive — the band for a key above sits at the top of the display.
**Rejected:** Flat 12-char grid (no spatial grouping), index-numeric order 0–11 (arbitrary).

---

## D-008 — Docs in flat /docs folder

**Decided:** Session 1
**Decision:** All six documentation files live in `docs/` at repo root, flat structure.
**Reason:** Simple, all cross-links are single-level, GitHub renders them natively.
**Rejected:** Nested `docs/spec/` + `docs/impl/` (unnecessary for current scale).

---

## D-009 — No bundler for v1

**Decided:** Session 1
**Decision:** Vanilla JS with ES Modules. No Vite, no Webpack, no build step.
**Reason:** GitHub Pages serves static files directly. Zero config to maintain.
**Rejected:** Vite (can be added in v2 without changing component structure if needed).

---

## D-010 — Project name is MapCopy

**Decided:** Session 1
**Decision:** The project is named MapCopy. MapCopy refers to the input engine specifically. MapCopy is the optimiser, trainer, and laboratory built around it.
**Reason:** Named by author.
**Rejected:** mapcopy-web (too narrow, implies only the web port).

---

## D-011 — All cost inferred from latency only, no geometric assumptions

**Decided:** Session 1
**Decision:** No hand assignment, no physical key spacing, no finger strength weighting. All ergonomic cost derived exclusively from measured latency.
**Reason:** Latency already encodes geometry — if two keys are far but fast for this user, they are cheap. Assumptions about anatomy introduce bias.
**Rejected:** Declared physical geometry as a cost input (adds assumptions, reduces personalisation).

---

## D-012 — Two latencies measured per pair: L1 and L2

**Decided:** Session 1
**Decision:** L1 = screen signal → first keydown (recognition + reach). L2 = first keydown → second keydown (transition). Both recorded and modelled separately.
**Reason:** L1 and L2 capture different cognitive and motor costs. Collapsing them into one number loses signal.
**Rejected:** Single inter-key latency only (loses recognition cost signal).

---

## D-013 — Rhythm measured via harmonic tempo sweep

**Decided:** Session 1
**Decision:** Presentation tempo sweeps progressively faster then slower per pair. Resonant frequency (most fluid tempo) is the cost metric. Breakdown point and recovery shape also recorded.
**Reason:** Treats the user as a dynamic system. Resonant frequency distinguishes motor-limited pairs (trainable) from cognitively-loaded pairs (avoid for frequent chars).
**Rejected:** Fixed-tempo trials with CV as rhythm metric (misses the frequency-response information).

---

## D-014 — Driver failsafe: hold Escape 3 seconds

**Decided:** Session 1
**Decision:** The OS-level driver restores system default layout if Escape is held for 3 seconds.
**Reason:** Prevents user lockout if a critical key was unmapped.
**Rejected:** No failsafe (too dangerous for a layout-remapping tool).

---

## D-015 — Output field is a native textarea

**Decided:** Session 1 end
**Decision:** The output area is a `<textarea>` driven programmatically in MapCopy mode, natively in regular mode.
**Reason:** All cursor movement, selection, scrolling, and editing behaviour comes free. Custom div-based output required reimplementing browser primitives badly.
**Rejected:** Custom div with span-based output (fragile, required reimplementing backspace/cursor/nav logic).

---

## D-016 — TypingArea owns prompt only, app.js owns dispatch

**Decided:** Session 1 end
**Decision:** `TypingArea.push(char)` / `.pop()` receives single printable chars only. All token handling, textarea writes, and routing live in `app.js`.
**Reason:** Previous versions had TypingArea trying to operate on the output field — wrong layer. Clean separation stopped the patching cycle.
**Rejected:** TypingArea handling special tokens (caused circular responsibility and repeated regressions).
